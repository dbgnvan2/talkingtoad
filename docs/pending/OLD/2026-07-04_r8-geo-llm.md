# R8 — LLM-driven GEO checks

`CENTRAL_CLAIM_BURIED`, `CHUNKS_NOT_SELF_CONTAINED`, `PROMOTIONAL_CONTENT_INTERRUPTS` were scored but
unimplemented (need an LLM to judge). Implemented in `api/services/geo_llm.py`.

## What changed
1. **`classify_geo_llm(text)`** → `{key: bool}` verdict via the existing AI router
   (`ai_analyzer._pick_model` / config-driven model — **no hardcoded model ID**, L1). Requests a
   strict JSON object; parses it (L4). **`_call_llm` never raises** — returns an error sentinel on
   failure.
2. **P14 guard** — `parse_geo_verdict` returns `{}` for an error/refusal string (`AI analysis
   skipped:` / `Error calling AI:` / `AI analysis failed:`) or any unparseable output, so a failed
   LLM call is **never** rendered as a finding. Only boolean verdict values are accepted.
3. **`geo_llm_issues(url, verdict)`** — emits the three codes with literal `make_issue` (removed
   from the dead-code allowlist).
4. **Endpoint** `POST /api/ai/geo-llm-checks` (auth + rate-limited) — **opt-in** (one LLM call per
   page; NOT run during every crawl). Re-fetches + parses the page (SSRF-guarded) for its body text,
   requires ≥200 words, then classifies.

## Cost / latency
One text LLM call per invocation (max_tokens=200, ~short). Opt-in per page → no per-crawl cost.

No scoring changes (impact derived via `_CALIBRATION`).

## Acceptance criteria → test (`tests/test_r8_geo_llm.py`)
- Structured-verdict parsing incl. prose-wrapped JSON + malformed → `{}` (L4).
- P14: error strings never parsed as content — `test_error_strings_are_not_content`,
  `test_classify_llm_error_yields_no_verdict`.
- Verdict → correct emitted codes — `test_issues_emitted_for_flagged_only`.
