# R7 — wire in the JS-render trio (Playwright)

`JS_RENDERED_CONTENT_DIFFERS`, `CONTENT_CLOAKING_DETECTED`, `UA_CONTENT_DIFFERS` were scored but
never emitted — `run_js_render_checks` had no non-test caller.

## What changed
1. **`js_render_issues(result)`** (`issue_checker.py`) — maps a `JSRenderResult`'s flags to issues
   with literal `make_issue`. Emits **nothing** if the render errored (Playwright absent / failed) —
   a failed render is never rendered as a finding.
2. **Engine step 7c** — post-crawl, gated on `HAS_PLAYWRIGHT` **and** `ai_readiness` enablement,
   capped at 10 pages/job; renders each and extends `all_issues`. One page's render failure never
   aborts the crawl.
3. **Dockerfile** — installs `playwright` + `chromium` (`playwright install --with-deps chromium`).
   **Cost:** ~400MB image growth + a headless-browser render per checked page (5s timeout each,
   capped 10/job). Omit the layer to keep the image small — the renderer degrades to a no-op.
4. Removed the three codes from the dead-code allowlist (they now emit via literal `make_issue`).

Keeps the existing SSRF guard (`js_renderer.run_js_render_checks` calls `is_ssrf_safe`).
**Follow-up (not in scope):** the rendered DOM could also feed `ORPHAN_PAGE` link discovery.

No scoring changes (impact derived via `_CALIBRATION`).

## Acceptance criteria → test (`tests/test_r7_js_render.py`)
- All flags → the three codes — `test_all_flags_emit_three_codes`.
- No flags → no issues — `test_no_flags_no_issues`.
- Errored / Playwright-absent render → no issues (graceful skip) — `test_errored_render_emits_nothing`.
