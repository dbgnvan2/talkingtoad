# Milestone 2.1: AIRouter Singleton (revised after QA hardening + audit)

## Goal
Establish a centralized routing layer for all AI provider calls to ensure identity-based key resolution and usage recording. Build the structural skeleton today; later milestones (M2.3 customer credentials, M2.5 usage tracking) plug into the stub points defined here.

## Scope of THIS cycle (skeleton-first, per Cycle Y user decision)

In scope:
- New module `api/services/ai_router.py` — singleton + interface + stub lookup/logging hooks.
- `AIResponse` dataclass, custom exception types.
- Provider driver interface + concrete drivers for OpenAI and Gemini (the two providers actually wired today).
- Refactor `api/services/rewriter.py` (the actual AI caller reached via the advisor router's `/api/ai/rewriter` endpoint — the spec's "advisor.py" reference was off-by-one; see §Reconciliation).
- Contract + unit tests for AIRouter behaviour.

Explicitly OUT of scope (deferred to later M2 sub-milestones):
- `customer_ai_credentials` table + Fernet encryption → M2.3.
- `ai_usage` table + persistence → M2.5.
- DeepSeek and Anthropic drivers → M2.9 (DeepSeek) and a future Anthropic milestone.
- Per-task-type model routing → M2.4.
- Refactor of `ai_analyzer.py` and the advisor service → follow-up cycle once AIRouter interface is stable.
- Per-customer identity model (where does `customer_id` come from on a request?) → would need its own M0.13 milestone; not blocking M2.1 because we always pass `SYSTEM_CONTEXT_ID` today.

## The contract

### Public interface

```python
SYSTEM_CONTEXT_ID = "system_account"   # used whenever no customer_id flows in

@dataclass(frozen=True)
class ModelConfig:
    model: str                 # e.g. "gpt-4o", "gemini-2.0-flash"
    max_tokens: int | None = None
    temperature: float = 0.2

@dataclass(frozen=True)
class AIResponse:
    content: str
    provider_id: str           # "openai", "gemini", ...
    model: str                 # echoes ModelConfig.model
    input_token_count: int
    output_token_count: int
    cost_estimate_usd: float   # 0.0 until M2.2 pricing table lands
    truncated: bool            # soft failure: response hit max_tokens
    # NOTE: hard failures raise; this dataclass is only constructed on success
    # or soft failure. See exceptions below.

class AIRouterError(Exception):
    """Base class for all AIRouter errors. Calling routers catch this."""

class ProviderAuthError(AIRouterError):
    """No customer key AND no system env key → 402 Payment Required."""

class ProviderAPIError(AIRouterError):
    """Provider returned a hard error (auth, rate limit, network)."""

class AIRouter:
    """Singleton entry point. Use the module-level `ai_router` instance."""

    async def call_text(
        self,
        *,
        customer_id: str,
        system_prompt: str,         # the "system" / instruction message
        user_prompt: str,           # the actual user content
        model_config: ModelConfig,
    ) -> AIResponse: ...

    async def call_vision(
        self,
        *,
        customer_id: str,
        system_prompt: str,
        image_bytes: bytes,
        image_mime: str,
        model_config: ModelConfig,
    ) -> AIResponse: ...

# Module-level singleton — implicitly thread-safe via import system.
ai_router = AIRouter()
```

### Reconciliation of QA hardening with audit reality

1. **`system_context` (QA spec) interpreted as `system_prompt`** — the system-role message passed to the LLM. (Not a correlation ID; not a tenant ID.) This matches the OpenAI / Gemini system_instruction concept already used in `rewriter.py:50`.
2. **`model_config` widened to a typed `ModelConfig` dataclass** rather than a free-form `dict`, to match the strictness of `AIResponse`.
3. **`error_flag` (QA spec) renamed to `truncated`** — hard failures raise, soft failures (max_tokens hit) set the flag. Resolves the internal inconsistency with the "no silent failures" rule.
4. **`AIResponse.usage_metadata`** (QA spec) **decomposed into named fields** (`input_token_count`, `output_token_count`, `cost_estimate_usd`) so the unit tests can assert concrete shapes. Aligns with PLAN-V3.0.md M2.1's `CompletionResult` dataclass.
5. **Singleton thread-safety: module-level instantiation only.** No lazy `__new__` / lock dance. FastAPI is async, not multi-threaded per worker; module load is the natural single-init point.
6. **Vision is a separate method** (per PLAN-V3.0.md M2.1) rather than a flag on `call()` — vision payloads, endpoints, and cost calcs all differ enough that one method per modality is clearer.

### Negative constraints (from QA spec, expanded for accuracy)

- **No keys cached in singleton state.** `_lookup_customer_credentials` returns the key for a single call; the AIRouter never holds it past method return.
- **No raw prompts in `log_usage`.** Hard whitelist enforced.
- **No SDK imports in *any* AIRouter caller.** The QA spec said "advisor.py" — but `api/routers/advisor.py` has no SDK imports today; the SDK calls live in the *service* layer. The real constraint is: after the refactor, `api/services/rewriter.py` MUST import only from `api.services.ai_router`. Forbidden imports: `openai`, `httpx` to provider URLs, `anthropic`, etc. (Enforced by an architecture test that greps the file.)
- **No silent failures.** Hard failures raise `ProviderAuthError` / `ProviderAPIError`. Soft truncation is flagged in `AIResponse.truncated`.
- **No global mutable state.** Module-level singleton init only. No setattr after construction.

## Acceptance criteria

1. `api/services/ai_router.py` exists and exposes the contract above.
2. `api/services/providers/openai.py` and `gemini.py` implement a shared driver interface and wrap existing HTTP call logic with token-count extraction (OpenAI returns `usage.prompt_tokens`/`completion_tokens`; Gemini returns `usageMetadata.promptTokenCount`/`candidatesTokenCount`).
3. `api/services/rewriter.py` is refactored to route via `ai_router.call_text(...)`. No direct provider imports remain. `RewriterResult` is adapted from `AIResponse` (preserves existing return shape for the router — no API surface change).
4. The four QA evaluator tests pass (fallback, auth failure, isolation/log-on-error, contract unification).
5. An architecture test asserts that `api/services/rewriter.py` does not import any provider SDK directly.
6. Full suite stays green: 1,307 tests passing (Cycle X baseline) + N new tests, 12 skipped, 0 failed.

## The evaluator (verbatim from QA, with one renamed for clarity)

1. **Unit Test 1 (Fallback):** customer has no key → system env key used → usage logged with `customer_id=SYSTEM_CONTEXT_ID`.
2. **Unit Test 2 (Auth Failure):** customer key missing AND system env key missing → `ProviderAuthError` raised. The advisor router catches this and returns HTTP 402.
3. **Unit Test 3 (Isolation / log-on-error):** mock the provider driver to raise `ProviderAPIError` mid-call → confirm `_log_usage` was called with `success=False` and the error message.
4. **Unit Test 4 (Contract):** swap provider driver in fixture (openai → gemini) → both return same-shape `AIResponse` for identical inputs.

Plus the architecture test in §AC#5, plus a contract test that the existing `/api/ai/rewriter` endpoint still returns the same JSON shape after the refactor (no frontend break).
