# GSC Connect / Property UX improvements

**Date:** 2026-07-06
**Status:** pending

## Problem

The GSC integration is single-tenant OAuth — the app acts as whatever Google
account completes Connect. The panel previously (a) never showed which account
was connected, (b) silently reused the browser's default Google account, and
(c) auto-selected `find(siteOwner) || properties[0]`, so it blindly picked a
property (often a `sc-domain:` property) the connected account was not verified
on, causing repeated 403s.

## Changes

1. **Force the account chooser on Connect** — `/connect` builds the
   authorization URL with `prompt="select_account consent"` so Google always
   shows the account picker.

2. **Permission-level labels + smart default (frontend)** — the property
   dropdown renders `{site_url} — {label}` where label maps
   `permission_level` (siteOwner→Owner, siteFullUser→Full,
   siteRestrictedUser→Restricted, siteUnverifiedUser→Unverified, else raw).
   The default-selected property is the one with the BEST permission
   (Owner > Full > Restricted > Unverified; ties keep original order). When the
   selected property is Unverified/Restricted, an amber hint warns that ingest
   may be denied.

3. **Show the connected account (best-effort)** — `SCOPES` gains `openid` and
   `userinfo.email`. In `/callback`, after `fetch_token`, the account email is
   derived from the id_token via `verify_oauth2_token`. The email fetch is fully
   wrapped in try/except: if anything fails, email is None and connect still
   succeeds and stores creds. `/status` returns `account_email` (str|None). The
   panel shows "Connected as {email}" or a subtle "account not identified —
   reconnect to show" when null.

## Scope / compatibility note

Change #3 adds `openid` + `userinfo.email` to the OAuth scopes. If the Google
OAuth consent screen for this install does not permit those scopes, Connect
could error — the owner should be advised. Existing connections keep working
(email shows null until the user reconnects once to populate it).

## Tests

- `tests/test_gsc_integration.py`:
  - `test_connect_forces_select_account` — authorization_url built with
    `prompt="select_account consent"`.
  - `test_status_includes_account_email` — stored email returned; no email → null.
  - `test_callback_stores_creds_when_email_fetch_raises` — email fetch raising
    still stores creds and returns 200.
- `frontend/src/components/__tests__/GSCInsightsPanel.test.jsx`:
  - dropdown options show permission labels;
  - Full property auto-selected over an Unverified first property;
  - limited-access hint renders for an unverified selection;
  - "Connected as {email}" renders when account_email present.
